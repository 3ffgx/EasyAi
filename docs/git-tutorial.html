📚 Git 基础概念

什么是 Git？

Git 是一个版本控制系统，可以帮你：

  • 记录代码的每一次修改
  • 随时回退到之前的版本
  • 多人同时开发同一个项目
  • 管理不同的功能分支

📂 Git 的三个区域

工作区
你编辑的文件
→
暂存区
git add 后
→
仓库
git commit 后

🔑 核心概念

概念 说明 类比
仓库 (Repository) 项目的 Git 数据库 项目的"档案室"
提交 (Commit) 一次代码修改的快照 保存一个"存档点"
分支 (Branch) 独立的开发线路 游戏的"多条剧情线"
合并 (Merge) 将分支的内容合到一起 把"支线剧情"汇入主线
远程 (Remote) 服务器上的仓库（如 GitHub） 云端的"备份"
记住这个流程：
编辑文件 → git add → git commit → git push

详细步骤

第一步：拉取最新代码

# 切换到主分支 git checkout main # 拉取远程最新代码 git pull origin main Already up to date.

第二步：创建功能分支

# 创建并切换到新分支 git checkout -b feature/添加用户头像 Switched to a new branch 'feature/添加用户头像'
分支命名规范：
• 新功能：feature/功能描述
• 修复 bug：fix/bug描述
• 文档更新：docs/更新内容

第三步：开发并提交

# 查看修改了哪些文件 git status modified: frontend/src/views/Profile.vue new file: frontend/src/components/Avatar.vue # 添加修改的文件 git add frontend/src/views/Profile.vue frontend/src/components/Avatar.vue # 提交，写清楚做了什么 git commit -m "feat: 添加用户头像上传功能" [feature/添加用户头像 a1b2c3d] feat: 添加用户头像上传功能 2 files changed, 150 insertions(+)

第四步：推送到远程

# 推送分支到远程 git push origin feature/添加用户头像 Enumerating objects: 5, done. Counting objects: 100% (5/5), done. ... * [new branch] feature/添加用户头像 -> feature/添加用户头像

第五步：在 GitHub 创建 Pull Request

  1. 打开 GitHub 仓库页面
  2. 点击 "Pull requests" → "New pull request"
  3. 选择你的分支
  4. 填写标题和描述
  5. 点击 "Create pull request"

第六步：合并后清理

# 切换回主分支 git checkout main # 拉取最新代码（包含你刚合并的） git pull origin main # 删除本地功能分支 git branch -d feature/添加用户头像 # 删除远程功能分支 git push origin --delete feature/添加用户头像

💻 常用 Git 命令

📋 日常操作

操作 命令 说明
查看状态 git status 查看哪些文件被修改了
查看修改 git diff 查看具体修改了什么内容
添加文件 git add 文件名 将文件添加到暂存区
添加所有 git add . 添加所有修改的文件
提交 git commit -m "说明" 提交暂存区的内容
推送 git push origin 分支名 推送到远程仓库
拉取 git pull origin 分支名 拉取远程最新代码

🌿 分支操作

操作 命令 说明
查看分支 git branch -a 查看所有分支
创建分支 git checkout -b 新分支名 创建并切换到新分支
切换分支 git checkout 分支名 切换到指定分支
合并分支 git merge 分支名 将指定分支合并到当前分支
删除分支 git branch -d 分支名 删除本地分支

📜 历史记录

操作 命令 说明
查看日志 git log --oneline 简洁显示提交历史
图形化日志 git log --graph --oneline --all 图形化显示分支历史
查看某次提交 git show 提交ID 查看某次提交的详细内容

⏪ 撤销操作

操作 命令 说明
撤销修改 git checkout -- 文件名 撤销对某个文件的修改
撤销 add git reset HEAD 文件名 将文件从暂存区移出
撤销 commit git reset --soft HEAD~1 撤销最近一次提交（保留修改）
慎用 git reset --hard：这会丢失所有未提交的修改！

🎯 实战场景

场景一：我要修复一个 Bug

# 1. 拉取最新代码 git checkout main && git pull # 2. 创建修复分支 git checkout -b fix/登录按钮不显示 # 3. 修复 bug，测试通过后提交 git add . git commit -m "fix: 修复登录按钮在移动端不显示的问题" # 4. 推送并创建 PR git push origin fix/登录按钮不显示

场景二：我要开发一个新功能

# 1. 拉取最新代码 git checkout main && git pull # 2. 创建功能分支 git checkout -b feature/暗黑模式 # 3. 开发功能，可以多次提交 git add frontend/src/styles/dark.css git commit -m "feat: 添加暗黑模式样式文件" git add frontend/src/views/Settings.vue git commit -m "feat: 设置页面添加暗黑模式开关" # 4. 推送 git push origin feature/暗黑模式

场景三：我的代码和别人的冲突了

# 1. 拉取最新代码时出现冲突 git pull origin main CONFLICT (content): Merge conflict in frontend/src/views/Chat.vue Automatic merge failed; fix conflicts and then commit the result. # 2. 打开冲突文件，找到冲突标记 # <<<<<<< HEAD # 你的代码 # ======= # 别人的代码 # >>>>>>> main # 3. 保留需要的代码，删除冲突标记 # 4. 标记冲突已解决并提交 git add frontend/src/views/Chat.vue git commit -m "merge: 解决 Chat.vue 的冲突"

场景四：我要撤销刚才的提交

# 撤销最近一次提交，但保留修改 git reset --soft HEAD~1 # 现在可以重新修改，然后重新提交 git add . git commit -m "新的提交说明"

场景五：我要临时切换分支处理其他事情

# 1. 暂存当前工作 git stash Saved working directory and index state WIP on feature/xxx: a1b2c3d ... # 2. 切换到其他分支处理紧急事务 git checkout main git checkout -b fix/紧急bug # 3. 处理完后切换回来 git checkout feature/xxx # 4. 恢复之前的工作 git stash pop
提交信息规范：
• 新功能：feat: 描述
• 修复 bug：fix: 描述
• 文档更新：docs: 描述
• 代码重构：refactor: 描述
